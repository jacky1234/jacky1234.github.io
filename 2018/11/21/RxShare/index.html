创建

RxJava2的5中观察者模式

类别 特点
Observable#Observer 能够发射0-n个数据，并以成功或错误时间终止
Flowable#subscriber 能够发射0-n个数据，并以成功或错误时间终止.支持背压，可以控制数据源发射的速度
Single#SingleObserver 只发射单个数据或错误事件
Completable#CompletableObserver 从来不发射数据，只处理onComplete和onError事件。可以看成Rx的runnbale
Maybe#MaybeObserver 能够发射0/1个数据，要么成功，要么失败。类似Optional

Observable的使用场景：

  1. 一般处理最大不超过1000条数据，而且几乎不会出现内存溢出
  2. 处理同步流

Flowable 较好的场景

  1. 处理以某种方式产生超过10KB的元素
  2. 文件读取与分析
  3. 读取数据库记录，也是一个阻塞和基于拉取模式
  4. 网络I/O 流
  5. 创建一个响应式非阻塞接口

Observable 分类

Hot Observable

  • 无论有没有观察者,事件始终都在发生。与订阅者的关系是1:n的关系，可以与多个订阅者共享信息。

Cold Observable

  • 只有观察者订阅了，才开始执行发射数据流，并且与订阅者是1:1的关系。当有多个不同订阅者，消息是完整重新完整发送的。 也就是说，事件是各自独立的。

Subject 和 Processor

public abstract class Subject<T> extends Observable<T> implements Observer<T>

分类

  • PublishSubject。once an {@link Observer} has subscribed, emits all subsequently observed items to the
    subscriber.
  • ReplaySubject。 Replays events to Observers.
  • AsyncSubject. An Subject that emits the very last value followed by a completion event or the received error to Observers.
  • BehaviorSubject. Subject that emits the most recent item it has observed and all subsequent observed items to each subscribed
Processor

能够支持背压，这是其与Subject的最大区别。
